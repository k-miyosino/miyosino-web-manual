# さくらインターネットデプロイ（本番）

> **現在のステータス:** 本番デプロイは一時停止中です（`deploy.yml` の `if: false`）。

## 概要

`main` ブランチへのpushでさくらインターネットへSSH経由でデプロイします。

## デプロイを有効化する手順

**手順1:** `miyosino-web` リポジトリの `.github/workflows/deploy.yml` を開きます。

**手順2:** `deploy-sakura` ジョブの `if: false` を以下のように変更します。

```yaml
# 変更前
if: false

# 変更後
if: github.ref == 'refs/heads/main'
```

**手順3:** さくらインターネットのSSH設定がGitHub Secretsに設定されていることを確認します。

| シークレット名 | 内容 |
|--------------|------|
| `SAKURA_SSH_HOST` | さくらインターネットのサーバーホスト名 |
| `SAKURA_SSH_USERNAME` | SSHログインユーザー名 |
| `SAKURA_SSH_KEY` | SSH秘密鍵（RSA等） |
| `SAKURA_DEPLOY_PATH` | デプロイ先ディレクトリパス |

## さくらインターネット管理画面

コントロールパネルへのアクセス情報は管理組合の引き継ぎ資料を参照してください。

## デプロイの仕組み

1. `npm run build` でNext.jsをビルド（`out/` ディレクトリに出力）
2. `scp` コマンドで `out/` の内容をさくらインターネットの指定ディレクトリへ転送
3. `chmod -R 755` でファイル権限を設定

## ロールバック手順

問題が発生した場合は、前のバージョンのファイルをさくらインターネットにアップロードします。  
または、前のコミットに戻してデプロイワークフローを再実行します。
