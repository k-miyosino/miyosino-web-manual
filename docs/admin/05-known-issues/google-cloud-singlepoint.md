# Google Cloud属人化課題

## 問題の概要

現在、Google Cloud（Google Places API）が**個人のGoogleアカウント**で管理されており、以下のリスクがあります。

- 担当者が退会・退任した場合にAPIが使用できなくなる
- 請求情報が個人のクレジットカードに紐づいている
- アカウント情報の引き継ぎが困難

## 現在の状況

| 項目 | 内容 |
|------|------|
| 使用サービス | Google Places API（New）|
| 用途 | 周辺施設データの取得・Kintoneへの同期 |
| 管理アカウント | ※担当者に確認してください |
| 請求先 | ※担当者に確認してください |

> **担当者:** アカウント情報はIT委員会の引き継ぎ資料を確認してください。

## 影響範囲

Google Places APIが利用できなくなった場合:
- `scripts/sync-places.ts` のデータ同期が失敗する
- **ホームページの「周辺施設」ページの更新が止まる**（既存データは表示されます）

## 推奨対応

### 短期対応
1. アカウント情報・パスワードをIT委員会内で共有
2. 引き継ぎドキュメントに記載（本ページ）

### 長期対応（推奨）
1. 管理組合法人名義のGoogleアカウントを作成する
2. Google Cloudプロジェクトを法人名義アカウントに移行する
3. 請求先を管理組合のクレジットカードに変更する

## アカウント移行手順（参考）

1. 新しいGoogle Cloudプロジェクトを法人アカウントで作成
2. 新プロジェクトでGoogle Places APIを有効化
3. 新しいAPIキーを発行
4. GitHub SecretsとWorkerのSecretsを新しいAPIキーに更新
5. 旧プロジェクトを削除（or 無効化）

## Google Places APIの料金

- 月$200のクレジットが無料で付与されます
- 周辺施設データのような小規模利用であれば通常無料枠内に収まります
- Google Cloud コンソールで請求状況を確認できます

## Cloudflareアカウントについて

同様の課題がCloudflareアカウントにも存在する可能性があります。  
Cloudflareのアカウント管理状況も確認・引き継ぎを行ってください。
